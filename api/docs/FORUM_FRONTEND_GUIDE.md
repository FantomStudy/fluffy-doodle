# Руководство по разработке фронтенда Форума

Это руководство предназначено для ИИ-ассистента или разработчика, который будет создавать клиентскую часть (React/Vue/Angular) для модуля форума.

## Стек технологий (рекомендуемый)
- **Framework:** React (Next.js) или Vue 3.
- **State Management:** TanStack Query (React Query) — идеально подходит для работы с этим API (кэширование, пагинация).
- **Styling:** CSS Modules или Tailwind CSS.

## Основные эндпоинты (Base URL: `/forum`)

### 1. Категории (`GET /categories`)
**Зачем:** Главная страница форума. Выводит список разделов (например, "Вопросы по Python", "Общий чат").
**Данные:** `id`, `name`, `description`, `topicsCount`.

### 2. Список тем (`GET /topics?categoryId=X`)
**Зачем:** Страница внутри категории.
**Важно:** Если `categoryId` не передан, выводит все последние темы.
**Отображение:** Для каждой темы показывайте:
- Заголовок (`title`).
- Автор (`author.fullName`) + Аватар.
- **Уровень автора:** Поле `author.stars` (используйте иконку звезды).
- Статус: `isSolved` (зеленая галочка, если тема решена).
- Просмотры (`views`) и ответы (`replies`).

### 3. Детали темы (`GET /topics/:id`)
**Зачем:** Страница обсуждения. Возвращает объект темы и массив `comments`.
**Логика решения:** 
- Если у комментария `isSolution: true`, выделите его рамкой или фоном "Лучший ответ".
- Кнопка "Отметить как решение" должна быть видна **только** если `currentUserId === topic.authorId`.

### 4. Создание контента
- `POST /topics` (нужна авторизация).
- `POST /topics/:id/comments` (нужна авторизация).

## Компонент "Карточка Пользователя" (Profile/Level)
В API форума каждый автор (`author`) возвращает:
```json
{
  "id": 1,
  "fullName": "Имя Фамилия",
  "avatar": "url",
  "stars": 150,
  "role": "student"
}
```
**UI Рекомендация:**
- 0-100 звезд: Бронзовая рамка.
- 101-500 звезд: Серебряная рамка.
- 500+ звезд: Золотая рамка.
Это позволит визуально выделить "опытных" пользователей, которые делятся решениями.

## Обработка ошибок
API возвращает стандартный формат:
```json
{
  "success": false,
  "error": "Описание ошибки на английском"
}
```

## Авторизация
Фронтенд должен отправлять куки `access_token` и `refresh_token` (HTTP-only), которые устанавливаются сервером при входе. Убедитесь, что в запросах (например, Axios) включен параметр `withCredentials: true`.
